# 20220725_doit타입스크립트프로그래밍_ch04_함수와메서드_내용정리

## 핵심 내용

```
- 함수의 매개변수 타입과 반환 타입을 생략하는 행위는 바람직하지 않다. 타입이 생략되면 함수의 구현 의도를 알기 어렵기 때문이다.
- 함수의 타입을 함수 시그니쳐(function signature)라고 하는데, 시그니쳐 형식은 "(매개변수1:타입, 매개변수2:타입) => 반환타입"의 형태를 가진다.
- type 키워드는 기존에 존재하는 타입을 단순히 이름만 바꿔서 사용할 수 있게 해주는데, 이러한 기능을 타입 별칭(type alias)라고 한다.
- 함수 매개변수에도 이름뒤에 물음표를 붙여서 선택적으로 매개변수를 사용하도록 만들 수 있는데, 이를 선택적 매개변수(optional parameter)라고 한다.
- 함수 표현식(function expression) 함수 선언문에서 함수 이름을 제외한 function(a, b) { return a + b } 과 같은 코드 부분을 의미한다.
- 일등 함수 : 함수와 변수를 구분하지 않는 특성을 의미한다.
- 함수 표현식을 담는 변수는 let 보다는 const 키워드로 선언하는 것이 바람직하다.
- 다중 패러다임 언어 (multi-paradigm language) : 실행문 지향 언어(execution-oriented language)와 표현식 지향 언어(expression-oriented language) 모두를 지원하는 언어를 의미한다
- 실행문은 return 키워드를 사용해야 CPU의 연산 결과를 알려주지만 표현문은 키워드를 안써도 알려준다는 차이가 있다.
- 콜백 함수 (callback function) : 일둥 함수 기능을 제공하는 언어에서 함수는 '함수 표현식'이라는 일종의 값으로 취급 받기 때문에 변수에 담아 쓸 수 있다. 이 말은 함수 표현식을 매개변수에도 담을 수 있다는 의미이며 이렇게 매개변수로 담아 쓰는 함수를 콜백 함수라고 부른다.
- 프레임워크(framework)는 프로그램의 전체 구조를 쉽게 작성할 수 있도록 설계한 라이브러리를 말한다.
- 프레임워크는 여러 프로그램이 공통으로 써야할 함수를 API로 제공하는데, 각 API는 공통되는 부분을 제외한 프로그램마다 개별적으로 구현할 수 있도록 지원할 필요가 있는데, 이런 부분을 콜백 함수로 집어 넣을 수 있도록 지원하면 API 구현 매우 유용하다.
- 고차 함수(high-order function) : 함수를 반환하는 함수를 의미한다. 함수형 언어에서의 함수 표현식은 값의 취급을 하기에 또 다른 함수를 반환할 수 있다.
- 디폴트 매개변수 (default parameter) : 함수 호출 시, 인수를 전달하지 않아도 매개변수에 기본 값을 설정할 때 쓰는 값
- 컴파일러가 중괄호 {} 를 객체로 해석하게 하려면 객체를 소괄호로 감싸주어야 한다.
- 타입스크립트는 클래스 속성 중 함수 표현식을 담는 속성은 function 키워드를 생략 가능하게 해준다.
- 메서드체인(method chain) : 객체의 메소드를 연속해서 호출하는 방식을 말한다.
```
