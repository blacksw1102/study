# 오브젝트 스캔 유형

## 테이블 스캔
- 인덱스를 거치지 않고 곧바로 디스크에 위치한 테이블에 접근하는 유형
- 유형: 테이블 풀 스캔

## 인덱스 스캔
- 인덱스로 테이블 데이터를 찾아가는 유형.
- 인덱스 범위 스캔, 인덱스 풀 스캔, 인덱스 고유 스캔, 인덱스 루스 스캔, 인덱스 병합 스캔

## 테이블 풀 스캔
- 인덱스를 거치지않고 테이블의 처음부터 끝까지 탐색하는 방식.
- WHERE 절에 인덱스를 안쓰거나, 전체 대비 과반의 데이터를 읽을 때 사용

## 인덱스 범위 스캔
- BETWEEN 구문 또는 LIKE 구문 사용 시 적용
- 좁은 범위를 스캔할 때 효율적으로 사용

## 인덱스 풀 스캔
- 테이블에 접근하지 않고 인덱스만 풀 스캔하는 방식.
- 인덱스를 구성하는 열 정보만 필요할 때 사용.
- 읽어오는 양이 적어서, 테이블 풀 스캔보다 성능상 유리

## 인덱스 고유 스캔
- WHERE 절에 = 조건으로 쓰이는 컬럼이 PRIMARY 또는 고유 인덱스일 경우 적용
- 인덱스 기반 스캔 방식 중 가장 효율정식 방식.

## 인덱스 루스 스캔
- 인덱스에서 필요한 부분만 골라서 스캔하는 방식
- 인덱스로 정렬된 상태에서 MAX(), MIN() 함수를 사용시 사용.
