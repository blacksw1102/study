# 20220726_doit타입스크립트프로그래밍_ch05_배열과튜플_내용정리

## 핵심 내용

```
- 자바스크립트에서 배열은 Array 클래스의 인스턴스, 즉 객체이다. 때문에 배열 객체를 위한 메서드를 제공한다.
- 타입스크립트에는 문자 타입이 없고 문자열의 내용을 변경할 수도 없다. 이런 특징 때문에 문자열 가공을 위해서는 먼저 문자열을 배열로 전환시켜야 한다.
- for...in문을 사용할 때, 배열의 경우에는 인덱스값을 순회하며, 객체의 경우 키값으로 순회한다.
- for...of문은 배열의 아이템값을 대상으로 순회한다.
- T와 같은 일종의 변(타입 변수)로 취급하는 것을 제네릭(generics)타입이라고 한다.
- 제네릭 형태로 구현된 함수는 "함수이름<타입변수>(매개변수)"의 형태로 호출해야 한다.
- 타입스크립트는 타입 변수가 생략된 제네릭 함수를 만나면 타입 추론을 통해 생략된 타입을 찾아낸다.
- 명령형 프로그래밍과 선언형 프로그래밍의 차이.
- 선언형 프로그래밍에서는 데이터의 생성과 가공을 분리하여 처리한다.
- 수학에서 map은 'x -> y' 형태로 어떤 값을 또 다른 값으로 만들어주는 연산을 의미한다.
- 타입스크립트에서 인터페이스, 클래스, 함수의 매개변수 등은 let이나 const 키워드 없어 선언하기에 const과 같은 효과를 주려면 readonly라는 타입 수정자가 필요하다.
- 전개 연산자를 이용하면 배열을 깊은 복사할 수 있다.
- 순수 함수로 구현하기 위해선 매개변수의 내용을 훼손하지 말아야 한다.
- 순수 함수를 고려하면 사실상 자바스크립트 배열이 제공하는 많은 메서드를 사용할 수 없다.
- 자바스크립트에서의 튜플은 따로 존재하지 않으며, 배열의 한 종류로 취급한다.
- 타입 별칭(alias)을 정의해서 any[] 타입의 객체의 의미를 명확하게 할 수 있다.
```
